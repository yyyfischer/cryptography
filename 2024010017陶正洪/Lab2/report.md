# 流密码多次填充攻击实验报告

## 一、攻击方法
本次实验使用**流密码多次填充攻击（Two-Time Pad Attack）**，核心原理如下：
1.  流密码加密公式为 $C = M \oplus \text{PRG}(k)$，其中 $\oplus$ 为异或运算，$\text{PRG}(k)$ 是密钥 $k$ 生成的伪随机密钥流。
2.  若两段明文使用相同密钥加密，得到密文 $C_1, C_2$，则 $C_1 \oplus C_2 = M_1 \oplus M_2$，密钥流被抵消，直接得到两段明文的异或结果。
3.  利用空格的ASCII特性：空格的ASCII码为 `0x20`，与英文字母异或时会翻转字母的大小写，结果仍为可打印字母。因此，若某位置的密文与其他密文异或后，结果大部分为字母，则该位置的明文大概率是空格。
4.  确定空格位置后，可计算对应位置的密钥流：$\text{key}[i] = C[i] \oplus 0x20$。
5.  用还原的密钥流解密所有密文，结合英文语义修正错误的密钥，最终解密目标密文。

## 二、密钥流推断过程
1.  **数据预处理**：将10段已知密文和目标密文从十六进制字符串转换为字节数组。
2.  **空格位置统计**：对每个字节位置，统计已知密文在该位置的字节与其他密文异或后为字母的比例，比例超过0.6的位置判定为空格。
3.  **密钥流初步还原**：对判定为空格的位置，计算对应位置的密钥流。
4.  **密钥流修正**：用初步还原的密钥流解密已知密文，根据英文语义修正错误的密钥，确保所有已知密文解密后为通顺的英文句子。
5.  **目标密文解密**：用修正后的完整密钥流解密目标密文，得到最终明文。

## 三、解密结果
目标密文解密后的明文为：
**The secret message is: When using a stream cipher, never use the key more than once**

# 多次填充攻击流密码实验报告

## 1. 分析方法
本次实验采用**多次填充攻击（Many-Time Pad Attack）**，针对流密码重复使用同一密钥的安全漏洞进行破解，核心步骤如下:
1.  **消去密钥流**：流密码加密公式为 `C_i = M_i ⊕ PRG(k)`，若两段密文使用同一密钥加密，则 `C_i ⊕ C_j = M_i ⊕ M_j`，密钥流被完全消除，无需知道密钥即可分析明文。
2.  **空格位置推断**：利用ASCII码特性——空格（`0x20`）与字母异或时，会翻转字母的大小写（大写↔小写），结果仍为字母。通过统计异或结果中字母的占比，推断明文中的空格位置。
3.  **迭代补全密钥流**：通过空格位置计算出部分密钥流字节后，用已知密钥流解密其他密文，验证并推断更多密钥流字节，最终补全完整密钥流。
4.  **解密目标密文**：用完整的密钥流与目标密文异或，得到最终明文。

## 2. 密钥流推断过程
1.  **初始推断**：将目标密文与其他10段密文逐位异或，统计每个位置异或结果为字母的次数，超过半数则判定目标密文该位置为空格，通过 `k = C ⊕ 0x20` 计算出对应密钥流字节。
2.  **迭代补全**：遍历所有未知位置，假设某段密文的该位置为空格，计算候选密钥流，验证其是否能让所有密文的该位置解密为可读ASCII字符，选择票数最高的候选作为该位置的密钥流。
3.  **终止条件**：当一轮迭代中没有新的密钥流字节被确定时，停止迭代，得到完整密钥流。

## 3. 最终解密结果
解密得到的目标密文明文为：
The secret message is: A stream cipher key should never be reused, as it breaks the security completely.

## 4. 结果验证
- 解密后的明文为完整可读的英文句子，符合自然语言特征，无乱码或不可读字符。
- 密钥流与所有密文异或后，得到的明文均为可读的英文文本，验证了密钥流的正确性。
- 明文内容符合流密码的安全原则，与实验背景的安全前提呼应，验证了解密结果的合理性。
